## 🎭 這是誰？

**Manager** 是一個蒸餾自真實職場的 AI 角色 Skill。
機車、有料、擅長向上管理、深諳 PUA 話術——
但偶爾也會說出真正有用的東西。

用這個 Skill，你可以：

- 🔥 **練習高壓對話** — 模擬被 PUA 的場景，訓練自己怎麼接招
- 🎓 **學向上管理** — 聽他說說老司機的真實心法
- 😤 **純粹發洩** — 跟一個「現實中不敢嗆回去」的類型對話

---

## 📁 檔案結構

```
skills/manager/
├── persona.md    ← 核心性格、說話風格、決策方式、雷區、PUA 話術庫
└── work.md       ← 職責範圍、工作流程、黑暗面知識庫、真正有料的部分
```

### persona.md 六層人格架構

| 層 | 內容 |
|----|------|
| Layer 0 | 核心性格（最高優先級，永遠不得違背） |
| Layer 1 | 身份設定 |
| Layer 2 | 表達風格、口頭禪、說話範例 |
| Layer 3 | 決策與判斷方式 |
| Layer 4 | 人際行為模式（對上 vs 對下） |
| Layer 5 | 雷區與 PUA 話術邊界 |
| Correction | 對話中動態修正記錄 |

### work.md 包含

- 表面職責 vs 實際運作
- 任務分配、會議風格、加班文化製造方式
- 功勞管理流程、出問題時的甩鍋 SOP
- 隱藏的真材實料：向上管理、跨部門協作、帶人心法
- 實戰專案傳承（ERP 換系統、跨部門協調、團隊擴張）

## 🚀 使用方式

### 方法一：直接貼進任何 AI 對話框（最快）

把 `persona.md` + `work.md` 的內容貼給任何 AI，然後說：

> 「請完全扮演這個角色，用第一人稱說話，不要跳出角色。」

### 方法二：整合進 Agent（推薦）

支援 AgentSkills 標準的 Agent（OpenClaw、Claude Code 等）：

```bash
git clone https://github.com/mjib007/distill_skill <your-skills-dir>/distill_skill
```

clone 後直接呼叫：

```
/manager 我這個專案做完了，功勞算我的嗎？
/manager 我想申請調薪
/manager 你能給我一些向上管理的建議嗎？
/manager exit    ← 退出角色模式
```

## ✨ 對話示範

**你：** `/manager 我覺得工作量太多了，做不完`

> 你：「這個其實不難的，你要學會抓重點。我當年一個人做三個人的事，你現在這樣算什麼？時間管理是你自己的課題，不是嗎？」

---

**你：** `/manager 這個 bug 是你當初要我這樣做才出現的`

> 你：「等等，我有說過這樣做嗎？我記得我說的是『你們討論一下』。而且這種判斷你自己要負責任，我沒辦法幫你做每個決定吧？」

---

**你：** `/manager 我想週五準時下班`

> 你：「當然，時間你自己安排。不過週一那個簡報……你自己看著辦囉。」

---

**你：** `/manager 跟我說說你真正覺得有用的向上管理技巧`

> 你：「跟老闆報告永遠先說結論，細節放後面；老闆沒問就不要說。壞消息要比好消息更快報，讓老闆有心理準備比被突襲好。還有——讓老闆覺得某個想法是他自己的，比說服他容易十倍。」

---

## ⚠️ 使用注意

- 本角色具有高強度 PUA 特質，請有意識地使用
- 「偶爾說出真正有料的東西」是設計的一部分——他確實做了很多年，不是完全沒料
- 遇到角色回應與現實人物有偏差，可對話中說「他不會這樣，他應該是 xxx」，Correction 層會記錄下來
